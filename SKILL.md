# 红浪漫群友天气推送技能

为微信群「红浪漫的兄弟姐妹们」生成每日早安天气预报推送，覆盖9个城市、11位群友。

## 数据获取策略（双数据源容错）

优先使用 nmc.cn（数据更丰富、结构化JSON、curl直接可用）。如果某个城市 nmc.cn 失败，自动切换到 weather.cma.cn 备用数据源。

### 主数据源：nmc.cn

- API地址：`https://www.nmc.cn/rest/weather?stationid={stationid}`
- 请求方式：`curl -s 'URL' --max-time 10`
- 返回格式：JSON
- 数据内容：实时温度、体感温度、湿度、风向风力、降水量、日出日落、预警、7天预报

cma.cn HTML 解析方法：
1. 在 HTML 文本中找到日期行（如 "27日（明天）"、"28日（后天）"），第一个通常是今天
2. 天气图标在 `class="weather weather-wX"` 中，wX 对应上方映射
3. 最高温在 `class="temp temp-high"` 的元素文本中
4. 最低温在 `class="temp temp-low"` 的元素文本中

### 数据提取流程

对每个城市执行：

1. 先 `curl -s 'https://www.nmc.cn/rest/weather?stationid={nmc_id}' --max-time 10`
2. 如果返回有效 JSON 且包含天气数据 → 用 nmc.cn 数据
3. 如果失败（超时/空数据/非200）→ 用 page_reader 获取 `https://weather.cma.cn/web/weather/{cma_id}.html`，从 HTML 中解析数据
4. 如果两个数据源都失败 → 跳过该城市

## 格式规则

1. **城市顺序固定**：北京、上海、广州、重庆、武汉、西安、南京、长沙、福州
2. **色标规则**：按最高温区分
   - 🔴 ≥35°C
   - 🟠 30~34°C
   - 🔵 ≤29°C
3. **emoji 规则**：晴☀️ 多云⛅ 阴☁️ 小雨🌧️ 中雨🌧️ 大雨⛈️ 雷阵雨⛈️ 雪❄️
4. **体感温度TOP**：取前三名，第1-2名用🔥，第3名用😓
5. **生活建议**：每人一条，结合实时天气、体感温度、湿度、降水量、预警信息、明天天气趋势给出贴心提醒
6. **温度建议规则**：35°C以上提防暑，低于16°C提示添衣保暖
7. **预警标注**：有预警信息时用⚠️标注
8. **禁止项**：不加粗、不标城市等级文字（如"高温""炎热"等）、不带风力和降水量数据
9. **分隔线**：━━━━━━━━━━━━━━ 固定不变，共16个━
10. **天气相同时省略“转”**：白天和夜间天气一致时只写一个天气，如 `☀️晴 20~32°C`；不同时才写 `转`，如 `⛅多云转阴 18~28°C`
11. **时段问候语**：根据当前推送时间自动选择问候语：00:00~11:59用"早安"，12:00~17:59用"午安"，18:00~23:59用"晚安"
12. **容错**：如果某个城市两个数据源都失败，跳过该城市
